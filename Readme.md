# iHub Africa Prototype

A modern, responsive landing page prototype for **iHub Africa**, showcasing digital skills programs, ecosystem partnerships, impact metrics, and student onboarding workflows.

## Author

**Kamogelo Digwamaje**

## Project Goals

This project serves as a collaborative front-end development initiative where contributors can:

* Build new features
* Improve UI/UX
* Fix bugs
* Optimize performance
* Experiment with new components

All changes should be submitted through feature branches and reviewed via Pull Requests before being merged into the main branch.

* HTML5

## Getting Started

Clone the repository:

```bash
git clone <repository-url>
cd ihub-africa-prototype
```

Run locally by opening `index.html` or using VS Code Live Server.

---

## Git Workflow

### 1. Create a Branch

Never work directly on `main`.

Create a feature branch:

```bash
git checkout -b feature/your-feature-name
```

Examples:

```bash
git checkout -b feature/navbar-improvements
git checkout -b feature/application-validation
git checkout -b fix/mobile-layout
```

---

### 2. Commit Changes

```bash
git add .
git commit -m "Add feature description"
```

---

### 3. Push Branch

```bash
git push origin feature/your-feature-name
```

---

### 4. Open a Pull Request

After pushing:

1. Open GitHub
2. Navigate to the repository
3. Click **Compare & Pull Request**
4. Submit your PR for review

---

## Branch Strategy

| Branch    | Purpose                       |
| --------- | ----------------------------- |
| main      | Production-ready code         |
| develop   | Integration branch (optional) |
| feature/* | New features                  |
| fix/*     | Bug fixes                     |
| docs/*    | Documentation updates         |

Examples:

```text
feature/dark-mode-enhancements
feature/partner-carousel
fix/form-validation
docs/readme-update
```

---

## Pull Request Requirements

Before opening a PR:

* Code builds successfully
* No console errors
* Responsive design maintained
* Existing functionality remains intact
* Meaningful commit messages used
